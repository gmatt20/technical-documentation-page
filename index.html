Block-Level Semantic HTML Elements:

<header>

Represents introductory content, such as navigational aids, logo, search form, or author name.

<nav>

Represents navigation links, such as menus, tables of contents, and indexes.

<aside>

Represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes.

<main>

Represents the dominant content of the body of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.

<article>

Intended to be independently distributable or resuable. Forum post, magazine/newspaper article, blog entry, podcast card, and widgets are some examples.

<section>

Represents a generic standalone section of a document. Sections should always have a heading.

CSS

Displays

The default page behavior is normal flow. Meaning that the page always starts out with either as blocks or inlines. The mindset when it comes to making websites it is that we should work with the normal flow, rather than struggling against it (Kevin Powell).


It is crucial to understand the basics; inline elements and block elements.


Inline Elements

Inline elements only use as much horizontal space as their content requires. Therefore, inline elements are not greedy. Inline elements do not start on a new line.


Block Elements

Block elements takes up the entire horizontal space it is allowed to get. Therefore, block elements not greedy and will take the full width of the screen. Block elements start on a new line.


Now, here are the other displays:


Flex

The flex property is used to create a flexible layout, which allows a natural responsive web design. Think of a flex container like a row or column of blocks that you can easily move around, resize, and align in different ways. This makes creating complex layouts much easier compared to traditional methods.


Inline-Block

The inline-block property allows an element to behave like an inline element while still retaining block element characteristics. They are laid out in a line, just like inline elements. They respect whitespace in the HTML, meaning spaces, tabs, and line breaks between inline-block elements in the source code will affect the rendered output.


Unlike purely inline elements, inline-block elements can have width and height set. They can have padding, margins, and borders applied, which will affect the layout just as with block elements.


You can place multiple inline-block elements next to each other on the same line, making it useful for creating layouts like navigation bars, buttons, or image galleries without the need for floats. It is useful for creating responsive layouts without using floats or flexbox/grid.


Grid

The grid property displays an element as a block-level grid container. The Grid Layout is a two-dimensional layout system, meaning it can handle both rows and columns. It allows you to define a grid structure and place items into this grid, providing more control over the alignment and spacing of elements compared to traditional layout methods like floats or flexbox.


Inherit

Inherits this property from its parent element.


None

The element disappears.

Box Model

The CSS box model describes the rectangular boxes that are generated for elements in the document tree and laid out according to the visual formatting model.

        width: 100px;
        height: 100px;
        margin: 10px;
        padding: 20px;
        border: 2px solid black;
        box-sizing: border-box;
    }

    JavaScript Skills Documentation

    1. Numbers and Math

    Basic arithmetic operations and using mathematical functions.

    let sum = 10 + 5; // 15 let product = 10 * 5; // 50 let squareRoot = Math.sqrt(16); // 4

    2. Strings

    Creating and manipulating strings using various methods.

    let greeting = "Hello, " + "World!"; // "Hello, World!" let upper = "hello".toUpperCase(); // "HELLO" let lower = "hello".toLowerCase(); // "hello" let template = `Hello, ${name}!`; // Template Literal

    3. console.log

    Outputting information to the browser console for debugging.

    console.log("Hello, World!");

    4. Variables

    Declaring variables using var, let, and const.

    var x = 10; let y = 20; const z = 30;

    5. Booleans and If-Statements

    Using boolean values and conditional statements.

    let isTrue = true; if (isTrue) { console.log("It's true!"); } else { console.log("It's false!"); }

    6. Functions

    Defining and calling functions.

    function greet(name) { return `Hello, ${name}!`; } console.log(greet("Alice"));

    7. Objects

    Creating and manipulating objects.

    let person = { name: "John", age: 30, greet: function() { console.log(`Hello, my name is ${this.name}`); } }; person.greet();

    8. Document Object Model (DOM)

    Selecting and manipulating DOM elements.

    // Selecting an element const element = document.querySelector('.my-element'); // Changing content element.textContent = "New Content"; // Adding a class element.classList.add('new-class'); // Event handling element.addEventListener('click', function() { console.log('Element clicked!'); });

    9. Loops

    Using for, while, and do...while loops to iterate over data.

    for (let i = 0; i < 5; i++) { console.log(i); }

    10. Arrays

    Creating and manipulating arrays using various methods.

    let fruits = ["apple", "banana", "cherry"]; fruits.push("date"); console.log(fruits); // ["apple", "banana", "cherry", "date"]

    11. JSON and Local Storage

    Understanding and using JSON, and storing and retrieving data using localStorage.

    // JSON let jsonString = JSON.stringify({ name: "Alice", age: 25 }); let jsonObject = JSON.parse(jsonString); // localStorage localStorage.setItem('user', jsonString); let storedUser = localStorage.getItem('user'); console.log(JSON.parse(storedUser));
