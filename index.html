CSS

Displays

The default page behavior is normal flow. Meaning that the page always starts out with either as blocks or inlines. The mindset when it comes to making websites it is that we should work with the normal flow, rather than struggling against it (Kevin Powell).


It is crucial to understand the basics; inline elements and block elements.


Inline Elements

Inline elements only use as much horizontal space as their content requires. Therefore, inline elements are not greedy. Inline elements do not start on a new line.


Block Elements

Block elements takes up the entire horizontal space it is allowed to get. Therefore, block elements not greedy and will take the full width of the screen. Block elements start on a new line.


Now, here are the other displays:


Flex

The flex property is used to create a flexible layout, which allows a natural responsive web design. Think of a flex container like a row or column of blocks that you can easily move around, resize, and align in different ways. This makes creating complex layouts much easier compared to traditional methods.


Inline-Block

The inline-block property allows an element to behave like an inline element while still retaining block element characteristics. They are laid out in a line, just like inline elements. They respect whitespace in the HTML, meaning spaces, tabs, and line breaks between inline-block elements in the source code will affect the rendered output.


Unlike purely inline elements, inline-block elements can have width and height set. They can have padding, margins, and borders applied, which will affect the layout just as with block elements.


You can place multiple inline-block elements next to each other on the same line, making it useful for creating layouts like navigation bars, buttons, or image galleries without the need for floats. It is useful for creating responsive layouts without using floats or flexbox/grid.


Grid

The grid property displays an element as a block-level grid container. The Grid Layout is a two-dimensional layout system, meaning it can handle both rows and columns. It allows you to define a grid structure and place items into this grid, providing more control over the alignment and spacing of elements compared to traditional layout methods like floats or flexbox.


Inherit

Inherits this property from its parent element.


None

The element disappears.



Box Model

The CSS box model describes the rectangular boxes that are generated for elements in the document tree and laid out according. to the visual formatting model.


The content box contains the actual content. Size it using inline-size and block-size, or:

width height

The padding box is the area around the content pox.

padding

The border box wraps the content and padding.

border

The margin box is the outermost layer which wraps the border, padding, and content.

margin

One more thing, in normal flow, the page uses content-box box-sizing. Content-box defines how the width and height of an element are calculated: should they include padding and borders, or not. The width and height you set on an element apply only to the content box. Padding, border, and margin are added outside of the content box.


The border-box model includes padding and border in the element's width and height. The width and height you set on an element include the content, padding, and border. Only the margin is added outside of this total width and height.


Always set border-box globally.


Responsive Design

Responsive web design makes web pages render well on a variety of devices and window or screen sizes.


Use flexible units (%, vh, vw, em, rem) for more responsiveness rather than using absolute units.


Extra small devices (600px and down)

@media only screen and (max-width: 600px) {...}

Small devices (600px and up)

@media only screen and (min-width: 600px) {...}

Medium devices (768px and up)

@media only screen and (min-width: 768px) {...}

Large devices (992px and up) (768px and up)

@media only screen and (min-width: 992px) {...}

Extra large devices (1200px and up)

@media only screen and (min-width: 1200px) {...}

Selectors

Selectors are used to target specific elements in the HTML to style. Selectors can target elements, classes, and IDs.


Here are 2 more advanced selectors:


::after inserts something after the content of each selected element.

::after

::before inserts something before the content of each selected element.

::before

These 2 selectors are paired with content: " "; to insert any content. Further, you can style the content like any normal element.


These 2 selectors great for adding more decorations (icons before or after a element without needing another element for it), adding quotes, displaying custom list bullets, create tooltips, etc.


There are also pseudo classes which is used to define a special state of an element. Here are user action pseudo-classes:


:hover activates when the mouse pointer points over the item.

:hover

:active activates when an item is being activated by the user. For example, when the item is clicked on.

:active

Next, here are functional pseudo-classes:


The matches-any pseudo-class matches any element that matches any of the selectors in the list provided.

:is()

The negation, or matches-none, pseudo-class represents any element that is not represented by its argument.

:not()

Finally, here are tree-structural pseudo-classes:


Represents an element that is the root of the document. In HTML this is usually the <html> element.

:root

Matches an element that is the first of its siblings.

:first-child

Matches an element that is the last of its siblings.

:last-child

JavaScript

1. Numbers and Math

Basic arithmetic operations and using mathematical functions.

let x = 10 + 5; // 15 let X-UA-Compatible = 10 - 5; // 5 let x = 10 * 5; // 50 let x = 10 / 5; // 2 let x = Math.sqrt(16); // 4

2. Strings

Creating and manipulating strings using various methods.

let greeting = "Hello, " + "World!"; // "Hello, World!" let upper = "hello".toUpperCase(); // "HELLO" let lower = "hello".toLowerCase(); // "hello" let template = `Hello, ${name}!`; // Template Literal

3. console.log

Outputting information to the browser console for debugging.

console.log("Hello, World!");

4. Variables

Declaring variables using var, let, and const.

var x = 10; let y = 20; const z = 30;

5. Booleans and If-Statements

Using boolean values and conditional statements.

let isTrue = true; if (isTrue) { console.log("It's true!"); } else { console.log("It's false!"); }

6. Functions

Defining and calling functions.

function greet(name) { return `Hello, ${name}!`; } console.log(greet("Alice"));

7. Objects

Creating and manipulating objects.

let person = { name: "John", age: 30, greet: function() { console.log(`Hello, my name is ${this.name}`); } }; person.greet();

8. Document Object Model (DOM)

Selecting and manipulating DOM elements.


Selecting an element

const element = document.querySelector('.my-element');

Selecting an ID

const element = document.getElementById("id");

innerHTML

const element = document.querySelector('.class').innerHTML;

Changing content

const element = document.querySelector('.class').innerHTML = "new content";

Adding a class

element.classList.add('new-class');

Event Handling

element.addEventListener('click', function() { console.log('Element clicked!'); });

JS has a unique feature that can react to events. Like when a user clicks on an HTML element, JS will execute some code. onclick = "" is the most common example of this. Here are more:


onmouseover = "" onmouseout = "" onmousedown = "" onmouseup = "" onmousemove = "" onmousemove = "" onkeydown = "" onkeypress = ""

Rather than always having to rely on HTML attributes, JS can also do the same within the JS code. JS can do this by using the: .addEventListener("event", function, useCapture);


9. Loops

Using for, while, and do...while loops to iterate over data.

for (let i = 0; i < 5; i++) { console.log(i); }

10. Arrays

Creating and manipulating arrays using various methods.

let fruits = ["apple", "banana", "cherry"]; fruits.push("date"); console.log(fruits); // ["apple", "banana", "cherry", "date"]

11. JSON and Local Storage

Understand storing and retrieving data using localStorage. Save data to local storage by:

localStorage.setItem('key', value);

Read data from local storage:

let lastname = localStorage.getItem(key);
